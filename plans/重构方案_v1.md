### 2.2 工具函数封装（data.js 导出）

```javascript
window.SkadrateData = {
  // 读取：合并 INITIAL_DATA 与 localStorage 数据
  load(),

  // 持久化：将当前内存数据写入 localStorage
  save(),

  // 增
  addItem(category, item),          // category: 'scintilla' | 'inlandEmpire' | 'gravityRainbow' | 'bSide'

  // 删
  deleteItem(category, id),

  // 改
  updateItem(category, id, newItem),

  // 查：获取最新 N 条（用于首页预览）
  getLatest(category, count = 4),

  // 查：获取全部（用于归档页）
  getAll(category),

  // 管理员检测
  isAdmin()
};
```

### 2.3 localStorage Key 设计

- 存储 Key: `skadrate_blog_v1`
- 结构：与 `INITIAL_DATA` 同型 JSON，管理员通过 CMS 面板增删改查后写入此处。
- 首页/归档页加载时：`const data = { ...INITIAL_DATA, ...localStorageData }`，以 localStorage 覆盖初始数据。

## 三、关键设计决策说明

| 决策点 | 方案 | 理由 |
|---|---|---|
| **多页面共享 data.js** | 每页通过 `<script src="../data.js">` 引入 | 无框架、无构建工具，原生 JS 共享最轻量 |
| **归档页路由** | 纯 HTML 文件（`pages/*.html`） | 无需路由库，静态托管友好 |
| **B-side 密码** | 前端硬编码 `0910`，输入正确后写入 `sessionStorage` 标记解锁状态 | 无后端时最简方案；归档页同逻辑 |
| **marked.js 集成** | `gravity-rainbow.html` 内联引入，笔记渲染时调用 `marked.parse()` | 轻量、无依赖 |
| **BGM 播放器** | 纯 CSS 绘制播放按钮 + `<audio>` 标签（预留 `src`） | 用户自行填入路径，规避自动播放限制 |
| **管理员模式** | URL 含 `?id=Skadrate` 时，`main.js` 检测并注入浮动「＋」按钮与发布面板 | 无后端时的伪 CMS 入口 |
